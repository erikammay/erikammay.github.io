SSL

Have you ever wondered if a web page is secure, or maybe even what it means to be secure? SSL (Secure Sockets Layer) is a type of encryption that is used to secure web pages/servers and networks.


TLS (Transport Layer Security) is a more modern version of SSL, but they are both very similar. They both secure web pages/servers and networks, which means they prevent attackers from accessing web pages and web servers that should not be open to the public.

If you have ever put your credit card information, phone number, or any other personal information, chances are, the website has an SSL certificate. SSL certificates allow a website to use SSL or TLS.



The main difference between TLS and SSL is the level of security. They are both cryptographic protocols that communicate between web servers and clients, but SSL is older, so all versions of it have been broken into. Four versions of TLS have been released and two have been broken into.



You may have heard of HTTPS and HTTP and thought, what's the difference? If you thought it was the level of security, you would be correct. HTTPS is more secure. The s stands for SSL, so since HTTP doesn’t have the SSL it is not very secure. If you’ve never heard or seen them before, they are at the beginning of URLs. If you have been on the world wide web, you have most likely seen it millions of times.

Having security on your website is very important and SSL or TLS will do that for you; especially TLS. A SSL certificate allows you to use SSL or TLS and HTTPS websites have the certificate.
